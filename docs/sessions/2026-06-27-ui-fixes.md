# Session 2026-06-27 — UI fixes and build errors

## Summary
Fixed frontend build errors, mobile post-dialog layout, chat input redundancy, SVG icon rendering, and keyboard handling on mobile.

## Build fixes
- **app.ts** — Fixed `<polyline>` SVG syntax (self-closing `/>` + `</polyline>`), replaced `#api.` with `api.` in template (Angular doesn't support native private fields), added public getters `get api()` and `get offlineDismissed()`
- **sticker-picker.ts** — `currentStickers()` → `currentStickers` (getter, not function)

## Mobile post-dialog layout fix
- **post-dialog.ts** — Restructured mobile bottom sheet from `overflow-y:auto` single container to `flex flex-col`: scrollable content (`flex-1 overflow-y-auto`) + fixed bottom action bar (`shrink-0`), used `85dvh` instead of `85%`, added `env(safe-area-inset-bottom)` for notched devices, changed `resize-y` to `resize-none` on textarea

## Chat input redundancy
- **chat.ts** — Removed separate image attach button (duplicated the "Фото" type in the message type menu). Moved the type selector button to the left of the text input (where attach button was). Applied the change to both desktop and mobile chat-input sections. Added `triggerFileInput()` call inside `selectType('image')` so selecting "Фото" opens the file picker.

## SVG icon rendering
- **chat.ts** — Added `DomSanitizer` with `bypassSecurityTrustHtml()` for message type icons. SVGs were being stripped by Angular's sanitizer when rendered via `[innerHTML]`. Now icons (Фото, Стикер, GIF, Опрос) render correctly in the dropdown.

## Keyboard handling on mobile
- **chat.ts** — Added `keyboardOpen` signal, `focus`/`blur` handlers on message inputs (both desktop and mobile). When input is focused: bottom nav is hidden (`document.body.classList.add('keyboard-open')`), chat container height changes to `100dvh - 3.5rem` (only top nav). On blur: restored to `100dvh - 7rem` (both navs). Used Angular `computed()` signal for dynamic height binding.
- **styles.css** — Added `.keyboard-open .bottom-nav { display: none }` to hide the fixed bottom navigation bar when keyboard is open.
