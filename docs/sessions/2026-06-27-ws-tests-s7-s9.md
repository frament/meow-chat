# WS Testing (T1-T12) + S7 Rate Limiting + S9 Friendship Check

## Goal
Стабилизировать WS-тесты (12 тестов по спецификации T1–T12), реализовать S7 (rate limiting) и S9 (friendship check) из ws-requirements.md.

## Changes

### Backend — `handlers.go`

- **Stop channel**: добавлен `stop chan struct{}` в `Handler`, метод `Close()`, `case <-h.stop: return` в `runHub` — чистое завершение hub goroutine между тестами
- **S7 Rate limiting**: sliding-window счетчик в `HandleWebSocket` — `msgTimestamps []time.Time`. При `len >= 10` за 1 секунду — `c.Close()`
- **S9 Friendship check**: перед INSERT проверка `SELECT 1 FROM friends WHERE ...` для `uid != toID`. Если не друзья → `{"type":"error","message":"not friends"}` + `continue`

### Backend — `ws_test.go` (new file, 15 tests)

- **Setup**: `setupWSTest()` + `setupWSDB()` — изолированный SQLite (DELETE journal) на каждый тест, 3 seeded users с friendship
- **Helpers**: `wsDial`, `wsDialDrain` (без `SetReadDeadline` — чинит баг fasthttp/websocket, где timeout ломает коннект), `wsWrite`, `wsRead`, `wsReadTimeout`
- **T1** — `TestWS_Handshake_JWT`: valid/missing/invalid/expired token
- **T2** — `TestWS_SendMessage`: send → DB check → echo
- **T2b** — `TestWS_SendMessage_Encrypted`: encrypted_content/iv в echo
- **T3** — `TestWS_PingPong_Alive`: 3s sleep → второе сообщение проходит
- **T4** — `TestWS_GracePeriod_Reconnect`: conn1.Close() → conn2 (same user) — `onlineUsers` stays true
- **T5** — `TestWS_PushCopy_Offline`: message to offline user → push_copy created
- **T5b** — `TestWS_PushCopy_NoCopyWhenOnline`: both online → no push_copy
- **T6** — `TestWS_MultiTab_Echo`: two tabs same user → both get echo, same msg id
- **T9** — `TestWS_MultiTab_NoDuplicate`: one tab → one echo, no second message
- **T10** — `TestWS_100ConcurrentConnections`: 100 conns, no panic
- **T12** — `TestWS_MassDisconnect_GracePeriod`: 10 conns → close all → onlineUsers stays true
- **S7** — `TestWS_RateLimit_Allowed`: 10 msgs with 150ms spacing → all pass
- **S7** — `TestWS_RateLimit_Exceeded`: 11 msgs burst → connection closed
- **S9** — `TestWS_Friendship_Required`: user 4 (no friends) → `"not friends"` error
- **HTTP** — `TestWS_HTTP401_NoToken`: HTTP GET /ws → 401

### Frontend — `app.ts`, `chat.ts`, `device-auth.ts`

- `device_approved` WS event handler (из предыдущей сессии)
- `visibilitychange → visible` reloads DM history (multi-tab catch-up)
